# Security Policy

## Supported Versions

Currently, this project is in active development. Only the latest version is supported with security updates.

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |
| < latest| :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability in this ID Card System, please follow these steps:

1. **Do not** open a public GitHub issue.
2. **Email** the project maintainer at: `your-email@example.com` (replace with your real email).
3. Alternatively, use the **private vulnerability reporting** feature on GitHub (if enabled for the repository).

You can expect an initial response within 48 hours. After that, I will work with you to understand and fix the issue. Once a fix is ready, I will release an update and publicly credit the reporter (unless you wish to remain anonymous).

### What to include in your report:
- A clear description of the vulnerability.
- Steps to reproduce it.
- Potential impact (e.g., data leak, XSS, authentication bypass).
- Any suggested fix (optional).

Thank you for helping keep this project secure.
